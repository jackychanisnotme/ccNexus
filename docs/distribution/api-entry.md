# AINexus API 入口白名单交付模板

## 适合谁

- 不想自己准备或维护上游，只想获得一个可用 `base_url` 和访问凭证的早期用户。
- 需要 Codex CLI、Claude Code 或 OpenAI SDK 快速接入的白名单用户。
- 能接受配额、限流、禁止用途、人工支持和小范围验证边界的用户。

## 交付内容

- 用户专属 `base_url`。
- 用户访问凭证或网关层认证信息。
- 可用模型列表和推荐客户端配置。
- 套餐额度、并发、速率和超额处理规则。
- 禁止用途、滥用封禁和异常请求处理规则。
- 基础用量统计：AINexus `/stats`、WebUI stats、上游账单交叉核对。

## 接入示例

```toml
model_provider = "AINexusAPI"
model = "gpt-5-codex"
preferred_auth_method = "apikey"

[model_providers.AINexusAPI]
name = "AINexusAPI"
base_url = "https://api.example.com/v1"
wire_api = "responses"
```

## 价格

- 封闭测试包：99-299 元/月，小额度、低并发、人工支持。
- Pro API 入口：299-999 元/月，更高额度、更高可用性、统计报表。
- 团队入口：999 元/月起，团队 base URL、部署隔离和服务支持。

## 交付时间

- 白名单审核：付款前或付款后 24 小时内确认。
- 账号开通：审核通过后 24 小时内。
- 额度调整：按支持渠道确认后手动调整。

## 限制

- 第一阶段仅小范围白名单，不公开大规模售卖。
- 不承诺无限额度、无限并发或永久可用。
- 不支持违反上游服务条款、刷量、滥用、转售和异常高并发用途。
- 成本无法核算或异常请求比例过高时，可以暂停或终止服务。

## 风控和成本核算

- 网关层记录用户、时间、路径、状态码、请求量和异常原因。
- AINexus WebUI 查看端点、凭证和时段统计。
- 上游账单每日核对，记录收入、上游成本和毛利。
- 触发封禁：短时间异常请求、明显刷量、共享凭证、违反禁止用途。

## 验收方式

- 用户拿到 base URL 和访问凭证。
- `/v1/models` 或兼容模型接口可返回可用模型。
- Codex CLI、Claude Code 或 OpenAI SDK 至少一个客户端跑通请求。
- 首日用量、错误率和上游成本有记录。
