# AINexus 6.3.1 Pro Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and install AINexus desktop version 6.3.1 on this Mac, using the current branch code and the same online-license public key as the license server, then overwrite the local `ainexus.app` bundle without restarting the running app.

**Architecture:** This release is a packaging-only update. The codebase already has the online-license client wired into desktop startup; the work is to keep that wiring intact, bump the desktop version metadata to 6.3.1, inject the server-matching public key at build time, and deploy the resulting macOS bundle over the existing app. Verification should prove the built app reports 6.3.1 and still resolves the embedded license key correctly.

**Tech Stack:** Go, Wails v2, macOS bundle signing, shell scripts, existing online-license client/service code.

---

### Task 2: Bump desktop release version to 6.3.1

**Files:**
- Modify: `cmd/desktop/wails.json`
- Modify: `cmd/desktop/build/darwin/Info.plist`
- Modify: `cmd/desktop/build/windows/wails.exe.manifest`
- Modify: `cmd/desktop/CHANGELOG.json`
- Modify: `cmd/desktop/CHANGELOG_CN.json`

- [ ] **Step 1: Update the desktop version metadata**

Set every desktop version field that still says `6.3.0` to `6.3.1`:

```json
{
  "info": {
    "productVersion": "6.3.1"
  }
}
```

and in the macOS plist:

```xml
<key>CFBundleVersion</key>
<string>6.3.1</string>
<key>CFBundleShortVersionString</key>
<string>6.3.1</string>
```

- [ ] **Step 2: Update the Windows manifest version to match**

```xml
<assemblyIdentity type="win32" name="AINexus" version="6.3.1.0" processorArchitecture="*"/>
```

- [ ] **Step 3: Update the top changelog entry text**

```json
{
  "version": "v6.3.1",
  "date": "2026-06-22",
  "changes": [
    "Release refresh for desktop build and online-license compatibility"
  ]
}
```

Use the same shape for both changelog files, keeping the rest of the history untouched.

- [ ] **Step 4: Verify the version strings are all 6.3.1**


Expected: no remaining `6.3.0` in those files.

### Task 4: Overwrite the installed app bundle and re-sign it

**Files:**
- Deploy to: `/Applications/ainexus.app`

- [ ] **Step 1: Replace the installed bundle with the new build output**

```bash
rsync -a --delete cmd/desktop/build/bin/ccNexus.app/ /Applications/ainexus.app/
```

- [ ] **Step 2: Re-sign the installed app bundle ad hoc**

```bash
codesign --force --deep --sign - /Applications/ainexus.app
```

- [ ] **Step 3: Verify the installed app reports the new version and has a valid signature**

```bash
plutil -p /Applications/ainexus.app/Contents/Info.plist | rg 'CFBundleShortVersionString|CFBundleVersion'
codesign -dv --verbose=2 /Applications/ainexus.app 2>&1 | sed -n '1,20p'
```

Expected: both version strings are `6.3.1`, and codesign reports the bundle is signed.
