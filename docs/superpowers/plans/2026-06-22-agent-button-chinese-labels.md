# Agent Button Chinese Labels Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Display `AI 助手` and `智能体配置` on the desktop header buttons when Simplified Chinese is selected.

**Architecture:** Keep the existing `t('agent.button')` and `t('agentProvider.button')` rendering flow. Add a source-level regression test for both locale files, then update only the two Simplified Chinese translation values.

**Tech Stack:** JavaScript ES modules, Node.js built-in test runner, Vite.

---

### Task 1: Localize the Agent Header Buttons

**Files:**
- Modify: `cmd/desktop/frontend/test/endpoint-modal-layout.test.js`
- Modify: `cmd/desktop/frontend/src/i18n/zh-CN.js`
- Verify unchanged: `cmd/desktop/frontend/src/i18n/en.js`
- Generated by build: `cmd/desktop/frontend/dist/`

- [ ] **Step 1: Write the failing translation test**

Add locale source reads near the existing frontend source fixtures:

```js
const zhSource = readFileSync(resolve(frontendRoot, 'src/i18n/zh-CN.js'), 'utf8');
const enSource = readFileSync(resolve(frontendRoot, 'src/i18n/en.js'), 'utf8');
```

Add this test:

```js
it('localizes the agent header buttons in Simplified Chinese', () => {
    assert.match(zhSource, /agent:\s*{[\s\S]*?button:\s*'AI 助手'/);
    assert.match(zhSource, /agentProvider:\s*{[\s\S]*?button:\s*'智能体配置'/);
    assert.match(enSource, /agent:\s*{[\s\S]*?button:\s*'AI Agent'/);
    assert.match(enSource, /agentProvider:\s*{[\s\S]*?button:\s*'Agent Provider'/);
});
```

- [ ] **Step 2: Run the focused test and verify RED**

Run:

```bash
cd cmd/desktop/frontend && node --test test/endpoint-modal-layout.test.js
```

Expected: FAIL because the Simplified Chinese source still contains `AI Agent` and `Agent Provider`.
